ARCHITECTURE NOTE 01 / CHOOSE STATE BY CONSISTENCY

State is a
design choice.

Start with the invariant: global read speed, relational queries, strong coordination, object durability, or an existing regional database.

INTERACTIVE SCENARIO
Choose a workload; the recommendation follows the consistency and data-shape requirement.
KVRead-heavy global config

Low-latency reads with eventual consistency. Good for flags, preferences, and cached responses—not transactional balances.

OPTIMIZE FOR READS
DURABLE OBJECTSOne authority per entity

Strongly consistent coordination for chat rooms, locks, rate limits, multiplayer state, and collaborative sessions.

OPTIMIZE FOR COORDINATION
D1Relational application data

Managed serverless SQL with SQLite semantics for structured metadata, accounts, and lightweight full-stack applications.

OPTIMIZE FOR SQL
R2Unstructured objects

S3-compatible, strongly consistent object storage with no egress fees for assets, uploads, backups, and datasets.

OPTIMIZE FOR BYTES
HYPERDRIVEKeep the existing database

Pool connections and accelerate PostgreSQL or MySQL. Disable query caching where read-after-write consistency is required.

ARCHITECTURE NOTE 02 / MOVE SPECIALIZED WORK OFF THE REQUEST PATH

The edge is a platform.
Not just a proxy.

Keep the request path short, then bind purpose-built services for background work, inference, retrieval, and browser automation.

ARCHITECTURE SIMULATION · NOT PROVISIONED BY THIS DEMO
01WorkerRespond now

Authenticate, route, validate, transform, and return the latency-sensitive result.

02QueuesProcess later

Decouple expensive work with at-least-once delivery; consumers must be idempotent.

03Workers AIRun inference

Invoke open-source models on serverless GPUs without provisioning inference infrastructure.

04VectorizeRetrieve context

Store and query embeddings for semantic search, recommendations, and RAG.

05Browser RunOperate the web

Control headless browsers for screenshots, PDFs, scraping, testing, and agent tasks.

Bindings are the connective tissue: the Worker receives platform capabilities through its environment instead of exposing service credentials to browser code.

ARCHITECTURE NOTE 03 / REASON THROUGH THE TRADE-OFF

Edge first.
Not edge only.

The strong answer is not “replace AWS.” It is to move the latency-sensitive request path first, then keep or migrate state deliberately.

  1. Move stateless routing, auth, and security policy to Workers.
  2. Keep the system of record and connect it with Hyperdrive when that reduces migration risk.
  3. Select KV, Durable Objects, D1, or R2 from consistency and data-shape requirements.
  4. Push long-running or failure-prone work through Queues; instrument both sides.
